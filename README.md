# ATLAS Alpha 0.0.1 Package A

## Estado actual

ATLAS ya cuenta con un core base estable orientado a eventos, un kernel ligero, un dispatcher, un runtime de servicios, procesos y pensamientos, además de un blackboard simple. El siguiente paso no es agregar visión, voz ni plugins, sino consolidar una capa cognitiva más robusta.

## Nuevo roadmap

### Fase 1: Cognitive Intelligence Layer

Se priorizan motores de razonamiento y decisión sobre módulos aislados:

- Executive Engine
- Goal Analyzer
- Task Decomposer
- Strategy Engine
- Planner
- Reasoner
- Evaluator
- Decision Engine

### Sprint 005: Workflow Engine

La arquitectura pasa de un orquestador monolítico a un motor configurable de workflows:

- Workflow
- Pipeline
- Stage
- Transition
- Executor
- Completion
- Metrics

Cada process podrá tener su propio workflow, por ejemplo:

- Goal
- Understand
- Retrieve Memory
- Reasoning
- Planning
- Validation
- Execution
- Reflection
- Learning
- Completed

### Fase 2: Cognitive Inspector

Más adelante se construirá un inspector cognitivo con enfoque de dashboard, no solo consola. La meta es exponer:

- procesos activos
- pensamientos en ejecución
- eventos por segundo
- CPU y memoria
- timeline de ejecución
- confianza del proceso

Este componente podrá evolucionar hacia una aplicación web sin cambiar la arquitectura base.

### Fase 3: Memory, Reasoning y Planning Engines

Una vez consolidado el workflow, se trabajará en motores más avanzados:

- Memory Engine con niveles de memoria: Working, Short-Term, Episodic, Semantic y Long-Term
- Reasoning Engine con deducción, inducción, abducción, analogía, resolución de restricciones y pruebas de hipótesis
- Planning Engine con task graph, priority queue, estimation de costos, dependency solver y execution planner
